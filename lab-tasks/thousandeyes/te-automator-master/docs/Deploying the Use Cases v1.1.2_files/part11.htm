Customer Website User Experience -‌

This section will detail the use of the ThousandEyes Automator to deploy an initial ThousandEyes environment designed to provide visibility into the experience of customer accessing a Customer-facing website. This deployment will include PageLoad and a DNS Trace test. Please play close attention to the detailed requirements aligned to each of the input fields.


Pre-requisites

The following data is required to complete the input fields in the Customer Website User Experience ThousandEyes Automator deployment page:

1. Account Group - an Account Group must be created, populated with the targeted Enterprise Agents and the Account ID, with the Account Admin role, prior to using the Automator

2. Cloud Agents - the Cloud Agent chosen by the customer based upon the user demographics.

3. Customer Website URL - the Public Facing URL of the Customer Website

4. Customer Website Domain Name - the Domain Name taken from the Customer Website URL

5. Test Interval - the test interval for running tests to the Customer Website URL and Domain Name. he choices are “Normal” - 5 minutes, “Critical” - 2 minutes or “Relaxed” - 10 minutes.


Deployment Inputs

Account Group Selection

Select the appropriate Account group from the dropdown. The targeted Organization and Account Group must be created prior to deployment as well as the Account ID used for authentication into the ThousandEyes Automator must have, at minimum, the Account admin role in the targeted Account Group.

For this deployment, we will be use the (Automator - Demo - Customer Website Demo) Account Group.


The second input field is for the ThousandEyes Cloud Agents being used to run the tests the ThousandEyes tests.The choice of agents is dependent upon the discovery performed with the customer of the specific locations that are representative of their customer demographic.

For this Demo, we will be using representative set of Cloud Agents in North America



The next 2 inputs are for the Customer Website Public URL and the Domain name of the URL.

For this Demo, we will be targeting the Customer website https://www.cisco.com. Each Cloud Agent will run a PageLoad test to the https://www.cisco.com and a DNS Trace test to the domain name cisco.com.



The final input is for the Test interval. For this Demo, we will choose “Relaxed” for both test intervals.

Now that we have all the inputs completed, it is important to re-check the inputs and, most importantly, check to make sure the Targeted Account Group is correct. Once you have confirmed the inputs you can deploy by clicking the DEPLOY button at the top of the page. You can follow the Deployment by scrolling towards the bottom of the page.


To check your deployment was successful, click on any of the items in the Deployment List and a nw tab will open attempting to access the targeted Account Group. (This may require you to enter your credentials to login).


Once in the ThousandEyes Platform, you can browse to check that all the tests, dashboard, reports, alert rules, credentials and labels have been created and are configured properly.




POST INSTALLATION ACTIVITIES

There are a few post-installation configuration items that are suggested following any ThousandEyes Automator deployment.


For the Customer Website User Experience use case please follow these steps to complete the installation.

1. Log in to the Account Group used for the deployment

2. Default Dashboard - change the default dashboard for the Account Group to Global Customer Website Status that was deployed by the Automator.

a. Choose Dashboards, click on the ThousandEyes Built-In dashboard label in the upper left hand corner, scroll to the Global Customer Website Status and select it

b. Once the Dashboard is active, click on the ellipse

c. Select to Edit Dashboard

d. Click on the checkbox for “Set as default account group”

e. Click Save Changes

3. Default Reports - change the default report for the Account Group to Global Customer Website Metrics that was deployed by the Automator.

a. Choose Dashboards, click on the ThousandEyes Built-In: Agent Alerts report label in the upper left hand corner, scroll to the Global Customer Website Metrics and select it

b. Once the Report is active, click on the ellipse

c. Select to Edit Report

d. Click on the checkbox for “Set as default account group”
