Microsoft Teams Comprehensive

This section will detail the use of the ThousandEyes Automator to deploy an initial ThousandEyes environment designed to provide visibility into a Microsoft Teams customer deployment. This deployment will include a Transaction Test that requires login credentials to Microsoft Teams. Please play close attention to the detailed requirements aligned to each of the input fields.


Pre-requisites

The following data is required to complete the input fields in the Microsoft Teams Comprehensive ThousandEyes Automator deployment page:

1. Agent installation - all Enterprise agents that will be used in the deployment must be installed and accessible in the targeted Account Group

2. Account Group - an Account Group must be created, populated with the targeted Enterprise Agents and the Account ID, with the Account Admin role, prior to using the Automator

3. Agents - the agents being deployed to. Choose either Cloud or Enterprise Agents.

4. Teams Username - the email account used to login to Microsoft Teams.

5. Teams Password - the password associated with the Teams Username

6. Transit Relay Test Interval - the interval for testing from the agent to world.tr.teams.microsoft.com. The choices are “Normal” - 5 minutes, “Critical” - 2 minutes or “Relaxed” - 10 minutes

7. Edge Node Transaction and DNS Trace Test Interval - the interval for the test Edge Node transaction and the DNS Trace Test. The choices are “Normal” - 10 minutes, “Critical” - 5 minutes or “Relaxed” - 15 minutes.


Once this data is obtained from the customer you can proceed with using the ThousandEyes Automator.


Deployment Inputs

Account Group Selection

Select the appropriate Account group from the dropdown. The targeted Organization and Account Group must be created prior to deployment as well as the Account ID used for authentication into the ThousandEyes Automator must have, at minimum, the Account Admin role in the targeted Account Group.

For this deployment we will be use the (Automator - Sandbox1 - Microsoft Teams - Comprehensive

- Demo) Account Group.




Next we need to select the Agents that the tests will be deployed to. The Agents shown in the list are directly related to those Cloud and Enterprise Agents the targeted Organization has access to. If all the Agents targeted for deployment are available in the dropdown you may proceed.

Choosing Agents that do not support Browser based tests will cause the deployment of one or more of the tests to fail.

The Enterprise Agents, that you earlier identified in the customer discovery as targets for this deployment, must be visible in the dropdown list. If the agents are not visible exit the Automator and work with the customer to get all the agents planned for the deployment installed prior to deployment.


Select the Agents by either scrolling through the list and selecting or by entering search criteria to narrow the list and then selecting the agent. The Agents will appear in the Selected Agents section. To remove any agent from the selected list simply click on the X next to the agent name.

For the demo, we will enter Branch in to the search criteria and then select the Branch Office 1 and Branch Office 2 agents.




The next 2 input fields are for Teams Username and Password. These should have been captured as part of the customer discovery process. These credentials will be stored in the Account Group Credentials store and are encrypted.

The username should be in the form of an email.


For this demo, we will choose out test account - tonystark@acmehero.onmicrosoft.com and then enter the password associated with the account.



The final 2 input fields are for test interval. The appropriate test interval should have been captured as part of the customer discovery process. We will choose “Relaxed” for both test intervals.



Now that we have all the inputs completed, it is important to re-check the inputs and, most importantly, check to make sure the Targeted Account Group is correct. Once you have confirmed the inputs you can deploy by clicking the DEPLOY button at the top of the page. You can follow the Deployment by scrolling towards the bottom of the page.



To check your deployment was successful, click on any of the items in the Deployment List and a new tab will open attempting to access the targeted Account Group. (This may require you to enter your credentials to login).



Once in the ThousandEyes Platform, you can browse to check that all the tests, dashboard, reports, alert rules, credentials and labels have been created and are configured properly.


POST INSTALLATION ACTIVITIES

There are a few post-installation configuration items that are suggested following any ThousandEyes Automator deployment.


For the Microsoft Teams End User Experience - Comprehensive use case please follow these steps to complete the installation.

1. Log in to the Account Group used for the deployment

2. Default Dashboard - change the default dashboard for the Account Group to Teams Global Status Dashboard that was deployed by the Automator.

a. Choose Dashboards, click on the ThousandEyes Built-In dashboard label in the upper left hand corner, scroll to the Teams Global Status Dashboard and select it

b. Once the Dashboard is active, click on the ellipse

c. Select to Edit Dashboard

d. Click on the checkbox for “Set as default account group”

e. Click Save Changes

3. Default Reports - change the default report for the Account Group to Teams Global Status Report that was deployed by the Automator.

a. Choose Dashboards, click on the ThousandEyes Built-In: Agent Alerts report label in the upper left hand corner, scroll to the Teams Global Status Report and select it

b. Once the Report is active, click on the ellipse

c. Select to Edit Report

d. Click on the checkbox for “Set as default account group”

e. Click Save Changes
