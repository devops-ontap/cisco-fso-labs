Microsoft 365 Comprehensive‌

This section will detail the use of the ThousandEyes Automator to deploy an initial ThousandEyes environment designed to provide visibility into a Microsoft 365 customer deployment. This deployment will include Transaction Tests that require Microsoft 365 Tenant name and login credentials with access to OneDrive, Sharepoint, Outlook, One Note and Office applications (Word, Excel or Powerpoint). Please play close attention to the detailed requirements aligned to each of the input fields.


Pre-requisites

The following data is required to complete the input fields in the Microsoft 365 Comprehensive ThousandEyes Automator deployment page:

1. Agent installation - all Enterprise agents that will be used in the deployment must be installed and accessible in the targeted Account Group

2. Account Group - an Account Group must be created, populated with the targeted Enterprise Agents and the Account ID, with the Account Admin role, prior to using the Automator

3. The Microsoft Tenant name - the Microsoft Tenant name must be provided by the customer. It is the lower-level domain. For example, for https://thousandeyes.sharepoint.com, thousandeyes is the lower-level domain or the Microsoft Tenant name.

4. Office Application - This input is for choosing (1) office application (Word, Excel. Powerpoint) to test to for all the agents.

5. MS365 Username - a user account in the form of an email address is required for most tests. The account needs to have access to all the Microsoft applications in the test regime (Sharepoint, OneDrive, OneNote, Outlook, Office Application)

6. MS365 Password - the password associated to the MS365 Username

7. Filename for Sharepoint Download - the filename, with extension, that will be targeted in the Sharepoint and OneDrive tests. The file should be copying into the root of the Documents folder in Sharepoint and in the MyFiles folder in OneDrive.

8. Agents - the agents being deployed to. Choose either Cloud or Enterprise Agents.

9. Microsoft 365 Test Interval - the test interval for all of the tests in this use case. The choices are “Normal” - 10 minutes, “Critical” - 5 minutes or “Relaxed” - 15 minutes.


Once this data is obtained from the customer you can proceed with using the ThousandEyes Automator.


Deployment Inputs

Account Group Selection

Select the appropriate Account group from the dropdown. The targeted Organization and Account Group must be created prior to deployment. In addition, the Account ID used for authentication into the ThousandEyes Automator must have, at minimum, the Account admin role in the targeted Account Group.

For this deployment we will be using the ( Automator - Sandbox1 - Microsoft 365 Comprehensive Sandbox) Account Group.


The second input is the Microsoft 365 Tenant Name. This would be the customer tenant name used to access certain Microsoft 365 components like OneDrive and Sharepoint.

For this Demo, our Microsoft 365 Tenant Name is ACMEHERO.


The next input is the choice of Office Application, Excel, Powerpoint or Word. The test will connect to the online application and login. For the Demo we will choose “Powerpoint”.


Next you must enter the Microsoft 365 account that will be used to login to the various components. It is best to use a test account. The account must have access to all the Microsoft 365 components.

We will choose our test account - tonystark@acmehero.onmicrosoft.com and then enter the password associated with the account.

Both of these entries will be entered into the ThousandEyes Credential Store associated with the targeted account group.



The next input field, “Filename for Sharepoint Download”, is associated with the Sharepoint and OneDrive transaction tests. The file name entered here (with extension) will be used in the 2 transaction tests as a sample download. The file used must be placed in the root of the Sharepoint Documents folder associated to the Microsoft 365 Account used as well as in the root directory of the My Files folder in OneDrive.

Enter the filename in the input field (we will be using the filename “Microsoft 365 Monitoring in a Nutshell v1.1.0.pdf” for the demo)





Next we need to select the Agents that the tests will be deployed to. The Agents shown in the list are directly related to those Cloud and Enterprise Agents the targeted Organization has access to. Since this is the Comprehensive option you must choose agents that are capable of running PageLoad and/or Transaction Tests.

Choosing Agents that do not support Browser based tests will cause the deployment of the tests to fail.

The Enterprise Agents, that you earlier identified in the customer discovery as targets for this deployment, must be visible in the dropdown list. If the agents are not visible exit the Automator and work with the customer to get all the agents planned for the deployment installed prior to deployment.

If all the Agents targeted for deployment are available in the dropdown you may proceed.

Select the Agents by either scrolling through the list and selecting or by entering search criteria to narrow the list and then selecting the agent. The Agents will appear in the Selected Agents section. To remove any agent from the selected list simply click on the X next to the agent name.

For the demo, we will enter Branch in to the search criteria and then select the Branch Office 1 and Branch Office 2 agents.



The final input field is the Test Interval. The choices for this field are “Normal” - 10 minutes, “Critical” - 5 minutes and “Relaxed” - 15 minutes. Choose the appropriate interval based upon customer requirements around alert notification interval vs. cost of test.

For the demo, we will choose “Relaxed” - 15 minutes


Now that we have all the inputs completed, it is important to re-check the inputs and, most importantly, check to make sure the Targeted Account Group is correct. Once you have confirmed the inputs you can deploy by clicking the DEPLOY button at the top of the page. You can follow the Deployment by scrolling towards the bottom of the page.



To check your deployment was successful, click on any of the items in the Deployment List and a new tab will open attempting to access the targeted Account Group. (This may require you to enter your credentials to login).



Once in the ThousandEyes Platform, you can browse to check that all the tests, dashboard, reports, alert rules, credentials and labels have been created and are configured properly.


POST INSTALLATION ACTIVITIES

There are a few post-installation configuration items that are suggested following any ThousandEyes Automator deployment.


For the Microsoft 365 End User Experience - Comprehensive use case please follow these steps to complete the installation.

1. Log in to the Account Group used for the deployment

2. Default Dashboard - change the default dashboard for the Account Group to Microsoft 365 Global Status that was deployed by the Automator.

a. Choose Dashboards, click on the ThousandEyes Built-In dashboard label in the upper left hand corner, scroll to the Microsoft 365 Global Status and select it

b. Once the Dashboard is active, click on the ellipse

c. Select to Edit Dashboard

d. Click on the checkbox for “Set as default account group”

e. Click Save Changes

3. Default Reports - change the default report for the Account Group to Microsoft 365 Global Status that was deployed by the Automator.

a. Choose Dashboards, click on the ThousandEyes Built-In: Agent Alerts report label in the upper left hand corner, scroll to the Microsoft 365 Global Status and select it

b. Once the Report is active, click on the ellipse

c. Select to Edit Report

d. Click on the checkbox for “Set as default account group”

e. Click Save Changes
