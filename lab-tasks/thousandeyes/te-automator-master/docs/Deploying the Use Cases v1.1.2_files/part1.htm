Deploying the Use Cases with the ThousandEyes Automator

This section of the document will detail the deployment of each use case using the ThousandEyes Automator. It will provide the information needed to perform customer-based discovery required for the data input fields in the Automator use case deployment page.

The ThousandEyes Automator was designed to take a pre-configured environment, built around a use case, and replicate it, while taking into account customer-centric requirements.

Three of the five use cases have 2 options for deployment - Comprehensive and Lite.

When to choose the Comprehensive vs Lite option

The Comprehensive option is designed for ThousandEyes deployments that include one or more Transaction or PageLoad tests.

The Lite option is designed for Network, HTTP Server and DNS test types and does not contain any Transaction or PageLoad tests.

There are 3 reasons to choose the Lite option vs. choosing the Comprehensive option:

1. Budgetary requirements - Transaction and PageLoad Tests are used to provide additional detail into the application performance of the targeted services. Do to this additional functionality, these tests consume more ThousandEyes units than non-browser based tests. A determination needs to be made on the use of Transaction or PageLoad tests for each customerâ€™s particular budgetary needs and monitoring requirements.

2. Customer requires greater depth of application testing - As stated above, Transaction and PageLoad Tests are used to provide additional detail into the application performance of the targeted services. In many cases, customer requirements will determine the level of application testing vs. network and http server access testing. For greater depth and additional detail around the actual experience of application use, Comprehensive tests would be the option. For more network-centric data, the Lite option would be preferred.

3. Unsupported appliances - Transaction and PageLoad tests require the ThousandEyes BrowserBot capability, which may not be available depending on the type appliance that the Enterprise agent is installed on. Appliances that do not support the BrowserBot include:

Do not deploy using the Comprehensive option to either of these 2 devices

- Cisco Catalyst 9K series with no SSD

- RaspberryPi

Customer Websites with and without CDN

In addition to the Comprehensive and Lite options there are also 2 options for the Customer Website Users Experience use case. The 2 options are based upon the method of accessing the Customer Website being targeted. Some Customer Websites are managed by Content Delivery Network provider and some are managed directly by the customer.

The Automator provides a different set of tests based upon these 2 options.


Once the determination is made on which option to choose, the customer specific information related to the option selected should be documented in preparation for deployment.
