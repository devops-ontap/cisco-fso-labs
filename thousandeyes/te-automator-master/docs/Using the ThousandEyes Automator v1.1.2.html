How to use the ThousandEyes Automator

This document describes how to use the ThousandEyes Automator to deploy initial environments that support these use cases. The recommended key use cases are as follows:

• Microsoft 365 End User Experience

• Microsoft Teams End User Experience

• Cisco Webex End User Experience

• Cisco SD-WAN Network Performance Visibility

• Customer Website User Experience


This document is associated with the following documents or tools:

• What is the ThousandEyes Automator?

• Deploying the Use Cases

• ThousandEyes Pricing Tool


What is the Automator?

The ThousandEyes Automator is a chrome extension that can be installed on Windows and Mac devices with Google Chrome installed. The ThousandEyes Automator intended use is for deployment of a prescribed set of tests, alert rules, labels, dashboards and reports that have been preconfigured for use with ThousandEyes Cloud and Enterprise agents is support of the recommended key use cases.


Accessing the Automator


The ThousandEyes Automator is a Chrome Browser extension that can be installed on Windows and Mac devices with Google Chrome installed. Once the Automator is installed using the Automator requires the following:

1. A Windows or Mac device with Google Chrome installed

2. A Cisco CCOID

3. A ThousandEyes ID and API Token

4. A ThousandEyes Organization and Account Group pre-allocated with the ThousandEyes ID having the role of (at minimum) Account Admin in the Account Group being deployed to

5. Access to SalesConnect.Cisco.com


You can access the Automator by:

1. Browsing to SalesConnect.Cisco.com

2. Login using your Cisco CCOID

Once logged in, insure the active tab is Salesconnect.Cisco.com and either:

1. Clicking on the Manage Extensions icon then scrolling to the ThousandEyes Automator

2. Clicking on the ThousandEyes icon pinned in the Extensions Taskbar

Using the Automator

• The Automator will load and present a dialog box to have the user enter their ThousandEyes ID and API Token



• If you don’t not have your API Token you can click on the “Find your API user token” link. A new tab will open where you will need to login ThousandEyes then you will be brought to your ThousandEyes profile page. The Basic Authentication token is located at the bottom of the page. Copy it and then paste it back into the dialog box.


• Once you have the ThousandEyes ID and Token in their respective input fields, click “Enter API Credentials:



• You will be presented with a screen validating your ThousandEyes platform login. To continue and open the Automator click “Start”.


• The ThousandEyes Automator homepage will open. There are links to the documents “What is the ThousandEyes Automator?”, “How to use the ThousandEyes Automator?” (this document) and What are the Uses Cases?. To proceed to the use cases and deploy, click “Go to the Use Cases”.



• You will be presented with the Use Case page. Four of the five use cases have 2 options for deployment. The Microsoft 365 , Microsoft Teams and Cisco Webex use cases all have 2 options that are listed as Comprehensive and Lite. The details of the difference and which to choose is based upon criteria outlined in the “What are the Use Cases?” Documentation. The Customer Website use case has 2 options based upon whether the use case involves access to the website via a CDN (Content Delivery Network). There is only one Cisco SD-WAN use case choice.

Once a use case (and option) has been determined, click on the appropriate tile to begin.


• To demonstrate how to use the ThousandEyes Automator, we will do a sample deployment one of the Use Cases. We will choose the “Microsoft 365 End User Experience - Lite” use case. Clicking on the Microsoft 365 End User Experience - Lite tile will open the Deployment page.

• For each Use Case, there will be customer specific inputs that will need to be filled in.

It is very important that any customer specific requirements for the input fields be gathered prior to using the Automator for deployment. These include installation of targeted Enterprise Agent, application specific information and test interval preference. For this example, we have gathered the necessary information, have created an Account Group to deploy to and have the Enterprise Agents installed and given access to the Account Group. Here is the what that looks like.


• Once the inputs are complete, you can deploy the by clicking the DEPLOY button. The Deployment progress can be followed by scrolling to the bottom of the page.


• Once the deployment is complete, simply click on any of the items in the deployment list and you will be brought into the ThousandEyes platform (if you not yet logged in or logged in under a different account you we need to log in with the account used to deploy using the ThousandEyes Automator.



• You now have a fully functioning ThousandEyes environment in support of the Microsoft 365 End User Experience - Lite use case.
