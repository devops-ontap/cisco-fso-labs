Cisco Webex Lite‌

This section will detail the use of the ThousandEyes Automator to deploy a initial ThousandEyes environment designed to provide visibility into a Cisco Webex customer deployment. This deployment will include HTTP Server, Network and DNS tests, Please play close attention to the detailed requirements aligned to each of the input fields.


Pre-requisites

The following data is required to complete the input fields in the Cisco Webex Lite ThousandEyes Automator deployment page:

1. Agent installation - all Enterprise agents that will be used in the deployment must be installed and accessible in the targeted Account Group

2. Account Group - an Account Group must be created, populated with the targeted Enterprise Agents and the Account ID, with the Account Admin role, prior to using the Automator

3. Webex Tenant Hostname - the domain name for the customer Webex account (i.e. cisco.webex.com)

4. Webex Collaboration Bridge - Primary and Secondary Data Center - choose the primary and secondary data center locations (can be found by executing the command - [ dig +trace “Webex Tenant Hostname”]). An associated Collaboration bridge server for each data center will be chosen for the tests.

5. Webex Multimedia Platform - Primary and Secondary Data Center - choose the primary and secondary data center locations (can be found by executing the command - [ dig +trace “Webex Tenant Hostname”]). An associated Multimedia Platform server for each data center will be chosen for the tests.

6. Agents - the agents to being deployed to. Choose either Cloud or Enterprise Agents.

7. DNS Trace Test Interval - the interval for testing DNS from the agent to the “Webex Tenant Hostname”. The choices are “Normal” - 10 minutes, “Critical” - 5 minutes or “Relaxed” - 15 minutes.

8. Webex Component Test Interval - the interval for the Webex components tests. The choices are “Normal” - 5 minutes, “Critical” - 2 minutes or “Relaxed” - 10 minutes.

Once this data is obtained from the customer you can proceed with using the ThousandEyes Automator.


Deployment Inputs

Account Group Selection

Select the appropriate Account group from the dropdown. The targeted Organization and Account Group must be created prior to deployment as well as the Account Id used for authentication into the ThousandEyes Automator must have, at minimum, the Account admin role in the targeted Account Group.

For this deployment, we will be use the Automator - Sandbox1 - Cisco Webex - Lite Demo Account Group.



The second input is the Webex Tenant Hostname. This would be the customer tenant name used to access their Webex environment.

For this Demo, our Webex Tenant Hostname is cisco.webex.com.


The next 4 inputs are related to the core components of Webex connectivity related to a customer’s specific environment. Each customer is assigned a Webex cluster which defines the 2 data centers that the customer’s environment is hosted in. There will be a primary and a secondary data center location defined for each customer. For the next 4 inputs, select the Primary or Secondary Data Center related your customer’s Webex environment for both the Collaboration Bridge and the Multimedia Platform.

For the Demo, the primary data center for the Cisco Webex Tenant is Dallas - dfw01 and the secondary data center is San Jose - sjc02




Next we need to select the Agents that the tests will be deployed to. The Agents shown in the list are directly related to those Cloud and Enterprise Agents the targeted Organization has access to. If all the Agents targeted for deployment are available in the dropdown you may proceed.

Select the Agents by either scrolling through the list and selecting or by entering search criteria to narrow the list and then selecting the agent. The Agents will appear in the Selected Agents section. To remove any agent from the selected list simply click on the X next to the agent name.

For the demo, we will enter Branch in to the search criteria and then select the Branch Office 4 and Branch Office 5 agents.

The final 2 input fields are for test interval. The appropriate test interval should have been captured as part of the customer discovery process. We will choose “Relaxed” for both test intervals.



Now that we have all the inputs completed, it is important to re-check the inputs and, most importantly, check to make sure the Targeted Account Group is correct. Once you have confirmed the inputs you can deploy by clicking the DEPLOY button at the top of the page. You can follow the Deployment by scrolling towards the bottom of the page.


To check your deployment was successful, click on any of the items in the Deployment List and a nw tab will open attempting to access the targeted Account Group. (This may require you to enter your credentials to login).




Once in the ThousandEyes Platform, you can browser to check that all the tests, dashboard, reports, alert rules, credentials and labels have been created and are configured properly.




POST INSTALLATION ACTIVITIES

There are a few post-installation configuration items that are suggested following any ThousandEyes Automator deployment.


For the Cisco Webex End User Experience - Lite use case please follow these steps to complete the installation.

1. Log in to the Account Group used for the deployment

2. Default Dashboard - change the default dashboard for the Account Group to Cisco Webex Global Status that was deployed by the Automator.

a. Choose Dashboards, click on the ThousandEyes Built-In dashboard label in the upper left hand corner, scroll to the Cisco Webex Global Status and select it

b. Once the Dashboard is active, click on the ellipse

c. Select to Edit Dashboard

d. Click on the checkbox for “Set as default account group”

e. Click Save Changes

3. Default Reports - change the default report for the Account Group to Cisco Webex Global Status that was deployed by the Automator.

a. Choose Dashboards, click on the ThousandEyes Built-In: Agent Alerts report label in the upper left hand corner, scroll to the Cisco Webex Global Status and select it

b. Once the Report is active, click on the ellipse

c. Select to Edit Report

d. Click on the checkbox for “Set as default account group”

e. Click Save Changes
