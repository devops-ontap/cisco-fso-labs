Cisco SD-WAN‌

This section will detail the use of the ThousandEyes Automator to deploy an initial ThousandEyes environment designed to provide visibility into a Cisco SD-WAN customer deployment. This deployment will include HTTP Server, Network and DNS tests, Please play close attention to the detailed requirements aligned to each of the input fields.

This use case requires detailed IP Address information of the Branch Site router and the Cisco SD- WAN vSmart Controller. This information must be obtained prior to use the ThousandEyes Automator for deployment.

Pre-requisites

The following data is required to complete the input fields in the Cisco SD-WAN ThousandEyes Automator deployment page:

1. Agent installation - all Enterprise agents that will be used in the deployment must be installed and accessible in the targeted Account Group

2. Account Group - an Account Group must be created, populated with the targeted Enterprise Agents and the Account ID, with the Account Admin role, prior to using the Automator

3. On-Prem Application URL - the URL of a targeted On Premise Application located in the data center. This URL needs to be provided by the customer.

4. Data Center Agent - the Enterprise Agent installed in the customer data center.

5. Branch Site Agent - the Enterprise Agent installed in the targeted Branch Site.

6. Branch Edge Router IP - Circuit #1 - the Public IP Address of the Edge Router in the Branch Site that is accessed via the primary circuit.

7. Branch Edge Router IP - Circuit #2 - the Public IP Address of the Edge Router in the Branch Site that is accessed via the backup circuit.

8. On-Prem Application Test Interval - the test interval for running test to the On-Prem Application. the choices are “Normal” - 10 minutes, “Critical” - 5 minutes or “Relaxed” - 15 minutes.

9. Overlay and Underlay Test Interval - the test interval run for both Overlay and Underlay testing. The choices are “Normal” - 5 minutes, “Critical” - 2 minutes or “Relaxed” - 10 minutes.

If there is only one circuit put the Primary Circuit in this input as well. This will be corrected in the post installation process.

Once this data is obtained from the customer you can proceed with using the ThousandEyes Automator.


Deployment Inputs

Account Group Selection

Select the appropriate Account group from the dropdown. The targeted Organization and Account Group must be created prior to deployment as well as the Account Id used for authentication into the ThousandEyes Automator must have, at minimum, the Account admin role in the targeted Account Group.

For this deployment, we will be use the Automator - Sandbox - Cisco SD-WAN Demo Account Group.



The second input field is the URL of an On-Premise Business application that is hosted in the targeted Data Center. Please input the entire URL. For this demo, the URL entered is https:// payroll.acmehero.com

The next 2 inputs are for the ThousandEyes Enterprise Agents being targeted in this deployment.

The Enterprise Agents, that you earlier identified in the customer discovery as targets for this deployment, must be visible in the dropdown list. If the agents are not visible exit the Automator and work with the customer to get all the agents planned for the deployment installed prior to deployment.

For the first input choose the Enterprise Agent that is located in the Data Center and for the second input choose the Enterprise Agent that is located in the targeted Branch Site.



The next 2 inputs are for the Public IPAddresses of the Edge Router at the Branch Site.

This deployment is designed for sites with 2 Circuits under SD-WAN management. If there is only 1 Circuit under management, simply put the same IPAddress in both inputs. Please follow the instructions in the post-deployment section to disable the extra test.


The next input is for the IPAddress of the Cisco vSmart Controller for your SD-WAN environment. Please refer to the Cisco SD-WAN Reference Architecture Guide found here for information on obtaining this IP Address.


The final 2 inputs are for the Test intervals. For this Demo, we will choose “Relaxed” for both test intervals.

Now that we have all the inputs completed, it is important to re-check the inputs and, most importantly, check to make sure the Targeted Account Group is correct. Once you have confirmed the inputs you can deploy by clicking the DEPLOY button at the top of the page. You can follow the Deployment by scrolling towards the bottom of the page.



To check your deployment was successful, click on any of the items in the Deployment List and a new tab will open attempting to access the targeted Account Group. (This may require you to enter your credentials to login).


Once in the ThousandEyes Platform, you can browse to check that all the tests, dashboard, reports, alert rules, credentials and labels have been created and are configured properly.




POST INSTALLATION ACTIVITIES

There are a few post-installation configuration items that are suggested following any ThousandEyes Automator deployment.


For the Cisco SD-WAN Network Performance Visibility use case please follow these steps to complete the installation.

1. Log in to the Account Group used for the deployment

2. Default Dashboard - change the default dashboard for the Account Group to Cisco SD-WAN Global Status that was deployed by the Automator.

a. Choose Dashboards, click on the ThousandEyes Built-In dashboard label in the upper left hand corner, scroll to the Cisco SD-WAN Global Status and select it

b. Once the Dashboard is active, click on the ellipse

c. Select to Edit Dashboard

d. Click on the checkbox for “Set as default account group”

e. Click Save Changes

3. Default Reports - change the default report for the Account Group to Cisco SD-WAN Global Status that was deployed by the Automator.

a. Choose Dashboards, click on the ThousandEyes Built-In: Agent Alerts report label in the upper left hand corner, scroll to the Cisco SD-WAN Global Status and select it

b. Once the Report is active, click on the ellipse

c. Select to Edit Report

d. Click on the checkbox for “Set as default account group”

e. Click Save Changes
