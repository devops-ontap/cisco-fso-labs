Microsoft 365 Lite‌

This section will detail the use of the ThousandEyes Automator to deploy an initial ThousandEyes environment designed to provide visibility into a Microsoft 365 customer deployment. This deployment will include HTTP Server and DNS tests and will require the Microsoft 365 Tenant name to target the OneDrive and Sharepoint customer-centric sites. Please play close attention to the detailed requirements aligned to each of the input fields.


Pre-requisites

The following data is required to complete the input fields in the Microsoft 365 Lite ThousandEyes Automator deployment page:

1. Agent installation - all Enterprise agents that will be used in the deployment must be installed and accessible in the targeted Account Group

2. Account Group - an Account Group must be created, populated with the targeted Enterprise Agents and the Account ID, with the Account Admin role, prior to using the Automator

3. The Microsoft Tenant Name - the Microsoft Tenant name must be provided by the customer. It is the lower-level domain. For example, for https://thousandeyes.sharepoint.com, thousandeyes is the lower-level domain or the Microsoft Tenant name.

4. Agents - the agents being deployed to. Choose either Cloud or Enterprise Agents.

5. DNS Trace Test Interval - the interval for testing DNS from the agent to microsoft.com. The choices are “Normal” - 10 minutes, “Critical” - 5 minutes or “Relaxed” - 15 minutes.

6. Microsoft 365 Test Interval - the test interval for all of the Microsoft component tests in this use case. The choices are “Normal” - 5 minutes, “Critical” - 2 minutes or “Relaxed” - 10 minutes.


Once this data is obtained from the customer you can proceed with using the ThousandEyes Automator.


Deployment Inputs

Account Group Selection

Select the appropriate Account group from the dropdown. The targeted Organization and Account Group must be created prior to deployment as well as the Account ID used for authentication into the ThousandEyes Automator must have, at minimum, the Account admin role in the targeted Account Group.

For this deployment we will be use the (Automator - Sandbox1 - Microsoft 365 Lite - Sandbox) Account Group. You will be targeting the Account Group created for your particular deployment.



The second input is the Microsoft 365 Tenant Name. This would be the customer tenant name used to access certain Microsoft 365 components like OneDrive and Sharepoint.

For this Demo our Microsoft 365 Tenant Name is ACMEHERO.



Next we need to select the Agents that the tests will be deployed to. The Agents shown in the list are directly related to those Cloud and Enterprise Agents the targeted Organization has access to. If all the Agents targeted for deployment are available in the dropdown you may proceed.

Select the Agents by either scrolling through the list and selecting or by entering search criteria to narrow the list and then selecting the agent. The Agents will appear in the Selected Agents section. To remove any agent from the selected list simply click on the X next to the agent name.

For the demo, we will enter Branch in to the search criteria and then select the Branch Office 4 and Branch Office 5 agents.




The final 2 input field are the Test Interval. The DNS Test Interval has the choice of “Normal” - 10 minutes, “Critical” - 5 minutes and “Relaxed” - 15 minutes. The Microsoft 365 Component Test Interval has the choice of “Normal” - 5 minutes, “Critical” - 2 minutes and “Relaxed” - 10 minutes. Choose the appropriate interval based upon customer requirements around alert notification interval vs. cost of test.

For the demo, we will choose “Relaxed” - 15 minutes


Now that we have all the inputs completed, it is important to re-check the inputs and, most importantly, check to make sure the Targeted Account Group is correct. Once you have confirmed the inputs you can deploy by clicking the DEPLOY button at the top of the page. You can follow the Deployment by scrolling towards the bottom of the page.



To check your deployment was successful, click on any of the items in the Deployment List and a new tab will open attempting to access the targeted Account Group. (This may require you to enter your credentials to login).


Once in the ThousandEyes Platform, you can browse to check that all the tests, dashboard, reports, alert rules, credentials and labels have been created and are configured properly.


POST INSTALLATION ACTIVITIES

There are a few post-installation configuration items that are suggested following any ThousandEyes Automator deployment.


For the Microsoft 365 End User Experience - Lite use case please follow these steps to complete the installation.

1. Log in to the Account Group used for the deployment

2. Default Dashboard - change the default dashboard for the Account Group to Microsoft 365 Global Status that was deployed by the Automator.

a. Choose Dashboards, click on the ThousandEyes Built-In dashboard label in the upper left hand corner, scroll to the Microsoft 365 Global Status and select it

b. Once the Dashboard is active, click on the ellipse

c. Select to Edit Dashboard

d. Click on the checkbox for “Set as default account group”

e. Click Save Changes

3. Default Reports - change the default report for the Account Group to Microsoft 365 Global Status that was deployed by the Automator.

a. Choose Dashboards, click on the ThousandEyes Built-In: Agent Alerts report label in the upper left hand corner, scroll to the Microsoft 365 Global Status and select it

b. Once the Report is active, click on the ellipse

c. Select to Edit Report

d. Click on the checkbox for “Set as default account group”

e. Click Save Changes
